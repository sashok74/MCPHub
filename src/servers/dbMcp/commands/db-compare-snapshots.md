# /db-compare-snapshots - Сравнение двух снимков запросов

Сравнивает результаты двух сохранённых снимков SQL-запросов и показывает различия.

## Использование

```
/db-compare-snapshots [имя1] [имя2]
```

## Что делает команда

1. **Загружает оба снимка** из локальной базы
2. **Сравнивает метаданные:**
   - Количество строк
   - Контрольные суммы (checksum)
   - Структуру колонок
3. **Выполняет построчное сравнение** данных
4. **Показывает различия** (если есть)
5. **Сравнивает производительность** (время выполнения)

## Примеры

### Пример 1: Снимки идентичны
```
/db-compare-snapshots before_opt after_opt
-> Результат сравнения:

ЭКВИВАЛЕНТНЫ
  - Строк: 1523 = 1523
  - Checksum: A1B2C3D4 = A1B2C3D4
  - Колонки: совпадают

Производительность:
  - before_opt: 245ms
  - after_opt: 89ms
  - Улучшение: 2.75x быстрее
```

### Пример 2: Снимки различаются
```
/db-compare-snapshots original modified
-> Результат сравнения:

РАЗЛИЧАЮТСЯ
  - Строк: 1523 != 1520
  - Checksum: A1B2C3D4 != X9Y8Z7W6

Различия (первые 10):
  Row 45: только в original  -> {"OrderID": 1234, "Status": 0}
  Row 46: только в original  -> {"OrderID": 1235, "Status": 0}
  Row 47: только в original  -> {"OrderID": 1236, "Status": 0}
```

### Пример 3: Разные колонки
```
/db-compare-snapshots v1 v2
-> Результат сравнения:

НЕСОВМЕСТИМЫЕ СТРУКТУРЫ
  - Колонки v1: OrderID, OrderDate, Status, Amount
  - Колонки v2: OrderID, OrderDate, Status
  - Отсутствует в v2: Amount
```

## Интерпретация результатов

| Индикатор | Значение |
|-----------|----------|
| ЭКВИВАЛЕНТНЫ | Данные полностью совпадают |
| РАЗЛИЧАЮТСЯ | Данные отличаются (показаны различия) |
| НЕСОВМЕСТИМЫЕ | Разная структура колонок |

## Workflow оптимизации

```
1. /db-snapshot original SELECT * FROM BigTable WHERE complex_condition...
2. Переписать запрос (оптимизация)
3. /db-snapshot optimized SELECT * FROM BigTable WHERE optimized_condition...
4. /db-compare-snapshots original optimized
   -> Эквивалентны, 3x быстрее
5. Применить оптимизированный запрос в коде
6. /delete-snapshot original && /delete-snapshot optimized
```

## Параметры

- **имя1** — имя первого снимка (baseline)
- **имя2** — имя второго снимка (для сравнения)

## Связанные команды

- `/db-snapshot` — создание снимка
- `/db-snapshots` — список всех снимков
- `/delete-snapshot` — удаление снимка

## Когда использовать

- **После оптимизации** — убедиться что результаты не изменились
- **При рефакторинге SQL** — проверить эквивалентность
- **Отладка** — найти где именно появились различия
- **Code review** — проверить что изменения в запросе корректны
