# /db-snapshot - Создание снимка результатов SQL-запроса

Выполняет SQL-запрос и сохраняет результаты как именованный снимок для последующего сравнения.

## Использование

```
/db-snapshot [имя] [SQL-запрос]
```

## Что делает команда

1. **Выполняет SQL-запрос** через dbmcp
2. **Сохраняет результаты** в локальную SQLite базу с:
   - Именем снимка
   - Текстом SQL-запроса
   - Количеством строк
   - Контрольной суммой (checksum)
   - Временем выполнения
   - Датой создания
3. **Возвращает информацию** о созданном снимке

## Примеры

### Пример 1: Снимок перед оптимизацией
```
/db-snapshot before_opt SELECT * FROM Orders WHERE Status = 1 ORDER BY OrderDate
-> Снимок создан:
  - Имя: before_opt
  - Строк: 1523
  - Checksum: A1B2C3D4
  - Время выполнения: 245ms
```

### Пример 2: Снимок после оптимизации
```
/db-snapshot after_opt SELECT o.* FROM Orders o WITH (INDEX(IX_Status)) WHERE o.Status = 1 ORDER BY o.OrderDate
-> Снимок создан:
  - Имя: after_opt
  - Строк: 1523
  - Checksum: A1B2C3D4
  - Время выполнения: 89ms
```

## Workflow оптимизации запросов

```
1. /db-snapshot original SELECT ... (исходный запрос)
2. Оптимизировать запрос (добавить индексы, переписать JOIN и т.д.)
3. /db-snapshot optimized SELECT ... (оптимизированный запрос)
4. /compare-snapshots original optimized -> убедиться что результаты идентичны
5. /db-snapshots -> просмотреть все снимки
6. Удалить снимки после завершения
```

## Параметры

- **имя** — уникальное имя снимка (например: `before_fix`, `v1`, `test_query`)
- **SQL-запрос** — любой SELECT запрос

## Ограничения

- По умолчанию сохраняется до 1000 строк (можно указать max_rows до 10000)
- Снимки хранятся в локальной SQLite базе
- Имена снимков должны быть уникальными

## Связанные команды

- `/db-snapshots` — список всех снимков
- `/compare-snapshots` — сравнение двух снимков
- `/delete-snapshot` — удаление снимка

## Когда использовать

- **Оптимизация запросов** — убедиться что оптимизированный запрос возвращает те же данные
- **Рефакторинг SQL** — проверить эквивалентность после изменений
- **Отладка** — сохранить состояние данных для анализа
- **Регрессионное тестирование** — сравнить результаты до и после изменений в БД
