# /db-explain - Анализ плана выполнения SQL-запроса

Получает план выполнения (execution plan) SQL-запроса без его фактического выполнения.

## Использование

```
/db-explain [SQL-запрос]
```

## Что делает команда

1. **Отправляет запрос** в SQL Server для анализа
2. **Получает план выполнения** в формате ShowPlanXML
3. **Анализирует ключевые метрики:**
   - Тип операций (Scan vs Seek)
   - Оценка строк (EstimateRows)
   - Стоимость операций (EstimatedTotalSubtreeCost)
   - Используемые индексы
   - Предупреждения оптимизатора
4. **Выдаёт рекомендации** по оптимизации

## Примеры

### Пример 1: Эффективный запрос
```
/db-explain SELECT * FROM Storage WHERE StorageID = 100
-> План выполнения:

  Эффективный план
  - Операция: Clustered Index Seek (Storage_PK)
  - Оценка строк: 1
  - Стоимость: 0.003
  - Индекс: Storage_PK (Clustered)
```

### Пример 2: Неоптимальный запрос
```
/db-explain SELECT * FROM Orders WHERE YEAR(OrderDate) = 2025
-> План выполнения:

  Проблемы обнаружены:
  - Операция: Clustered Index Scan (полное сканирование!)
  - Оценка строк: 50000
  - Стоимость: 2.45

  Рекомендации:
  - Функция YEAR() препятствует использованию индекса
  - Перепишите: WHERE OrderDate >= '2025-01-01' AND OrderDate < '2026-01-01'
```

### Пример 3: Отсутствующий индекс
```
/db-explain SELECT * FROM Transfers WHERE ContragentID = 500
-> План выполнения:

  Missing Index Warning:
  - SQL Server рекомендует создать индекс на ContragentID
  - Потенциальное улучшение: 85%

  CREATE INDEX IX_Transfers_ContragentID
  ON Transfers(ContragentID)
  INCLUDE (...)
```

## Ключевые метрики плана

| Метрика | Описание | Хорошо | Плохо |
|---------|----------|--------|-------|
| PhysicalOp | Тип операции | Index Seek | Table Scan, Index Scan |
| EstimateRows | Оценка строк | Малое число | Большое число |
| EstimatedTotalSubtreeCost | Общая стоимость | < 1 | > 10 |
| EstimateIO | Стоимость I/O | Низкая | Высокая |

## Типы операций (от лучшего к худшему)

1. **Clustered Index Seek** — точечный поиск по кластерному индексу
2. **Index Seek** — точечный поиск по некластерному индексу
3. **Index Scan** — сканирование индекса
4. **Clustered Index Scan** — сканирование таблицы
5. **Table Scan** — полное сканирование (heap)

## Workflow оптимизации

```
1. /db-explain SELECT ... (исходный запрос)
   -> Видим Table Scan, стоимость 5.2

2. Анализируем проблему:
   - WHERE по неиндексированному полю
   - Функция в WHERE (YEAR(), UPPER())
   - Неселективный предикат

3. Создаём/модифицируем индекс или переписываем запрос

4. /db-explain SELECT ... (новый запрос)
   -> Видим Index Seek, стоимость 0.01

5. /db-snapshot + /db-compare-snapshots для верификации результатов
```

## Интеграция с другими командами

| Команда | Когда использовать |
|---------|-------------------|
| `/db-explain` | Сначала — понять текущий план |
| `/db-snapshot` | После — сохранить результаты для сравнения |
| `/db-compare-snapshots` | В конце — убедиться что данные не изменились |

## Формат вывода

План возвращается в формате XML (ShowPlanXML). Claude автоматически:
- Парсит XML
- Извлекает ключевые метрики
- Форматирует для удобного чтения
- Выделяет проблемные места
- Даёт рекомендации

## Когда использовать

- **Перед оптимизацией** — понять текущее состояние
- **После оптимизации** — убедиться что план улучшился
- **Отладка медленных запросов** — найти узкие места
- **Code review** — проверить что запрос эффективен
- **Планирование индексов** — понять какие индексы нужны
