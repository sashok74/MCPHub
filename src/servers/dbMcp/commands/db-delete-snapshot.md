# /db-delete-snapshot - Удаление снимка запроса

Удаляет сохранённый снимок SQL-запроса из локальной базы.

## Использование

```
/db-delete-snapshot [имя]
```

## Что делает команда

1. **Находит снимок** по имени в локальной SQLite базе
2. **Удаляет снимок** и все связанные данные
3. **Подтверждает удаление**

## Примеры

### Пример 1: Удаление одного снимка
```
/db-delete-snapshot before_opt
-> Снимок 'before_opt' удалён
```

### Пример 2: Снимок не найден
```
/db-delete-snapshot nonexistent
-> Снимок 'nonexistent' не найден
```

## Workflow очистки

```
1. /db-snapshots -> посмотреть все снимки
2. /db-delete-snapshot old_test_1
3. /db-delete-snapshot old_test_2
4. /db-snapshots -> убедиться что удалены
```

## Массовое удаление

Для удаления нескольких снимков выполните команду несколько раз:

```
/db-delete-snapshot snapshot1
/db-delete-snapshot snapshot2
/db-delete-snapshot snapshot3
```

## Связанные команды

- `/db-snapshot` — создание снимка
- `/db-snapshots` — список всех снимков
- `/db-compare-snapshots` — сравнение снимков

## Когда использовать

- **После завершения оптимизации** — очистить временные снимки
- **Освобождение места** — удалить устаревшие снимки
- **Исправление ошибок** — удалить снимок с неправильным именем
- **Перед повторным использованием имени** — освободить имя для нового снимка

## Важно

- Удаление необратимо
- Данные снимка будут потеряны
- Рекомендуется сначала выполнить `/db-snapshots` чтобы убедиться в правильности имени
