# /pm-research - Полный цикл исследования сущности

Автоматическое исследование сущности (таблицы, модуля, бизнес-процесса) с заполнением project-memory.

## Использование

```
/pm-research [тема]     # Тема: название таблицы, бизнес-термин, модуль
```

## Алгоритм выполнения

### Фаза 1: Что уже знаем? (project-memory)

Вызови `prepare_context(task_description="$ARGUMENTS")` — один вызов покрывает всё:
- Все 7 хранилищ PM (facts, forms, processes, queries, glossary, relationships, patterns)
- Batch enrichment топ-entities (facts + relationships + forms уже включены)
- Gap identification со suggested_actions

Из результатов определи **целевые сущности** (1-3 основных из `entities[]`).

**Запомни** текущее покрытие из `entities[].coverage` (before) для финального отчёта.

**Если покрытие = `full` для всех сущностей** -> перейди сразу к Фазе 5 (только отчёт, без исследования).

### Фаза 2: Исследование БД (dbmcp)

Для каждой целевой сущности **где покрытие < full**:

**Шаг 2.1 — Найти таблицы** (если имя таблицы неизвестно):
```
dbmcp: list_objects(name_pattern='%EntityName%')
```

**Шаг 2.2 — Схема и связи** (параллельно):
```
dbmcp: get_table_schema(table='EntityName')
dbmcp: get_table_relations(table='EntityName')
dbmcp: get_table_sample(table='EntityName', limit=5)
```

**Шаг 2.3 — Связанные таблицы** (если есть дочерние/detail таблицы):
Из get_table_relations найди таблицы-детали (incoming FK).
Для основных (до 3) выполни:
```
dbmcp: get_table_schema(table='DetailTable')
```

**Шаг 2.4 — Хранимые процедуры/функции** (если есть):
```
dbmcp: search_object_source(pattern='%EntityName%')
```
Если найдены — получи определения ключевых (до 2):
```
dbmcp: get_object_definition(object_name='ProcName')
```

### Фаза 3: Исследование кода (C++)

**Шаг 3.1 — Формы и диалоги**:
```
Grep: 'EntityName' в *.h с glob='*Dlg.h' или '*ViewForm*.h'
```
Если найдены — прочитать заголовочный файл (Read) для понимания структуры класса.

**Шаг 3.2 — SQL-запросы**:
```
Grep: 'EntityName' в *.h с glob='*_SQL.h'
```
Если найден SQL-файл — прочитать для понимания основных операций (SELECT/INSERT/UPDATE/DELETE).

**Шаг 3.3 — Бизнес-логика** (опционально, если есть отдельный класс):
```
Grep: 'class C.*EntityName' в *.h
```
Если найден класс (не Dlg, не ViewForm) — прочитать заголовок для понимания бизнес-методов.

**ВАЖНО**: Не читай больше 3-4 файлов. Цель — получить обзор, а не глубокий анализ.

### Фаза 4: Сохранение в project-memory

На основе собранной информации сохрани **всё за один раз** через `bulk_save`:

**facts** (обязательные типы):
- `schema` — структура таблицы (ключевые колонки, PK/FK, типы данных)
- `business_rule` — бизнес-правила (статусы, типы, валидации, enum-значения)
- `relationship` — текстовое описание связей с другими таблицами
- `code_pattern` — если обнаружен паттерн в коде (наследование, интерфейсы)
- `gotcha` — если обнаружены неочевидные особенности (необычные имена колонок, нестандартные FK)

**glossary** — термин + синонимы + контекст + menu_path (если известен)

**relationships** — структурированные связи (entity_from, entity_to, rel_type):
- `creates` — A создаёт B
- `contains` — A содержит B (header -> detail)
- `references` — A ссылается на B (FK)
- `depends_on` — A зависит от B
- `produces` — A порождает B (результат процесса)
- `consumes` — A потребляет B

**form_mappings** — если найдены формы:
- template, class_name, file_path, main_table, operations

После bulk_save:
- `save_module(module_name, entities)` — привязать к модулю если определён
- `set_entity_status(entity, coverage)` — установить новый уровень покрытия:
  - `schema_only` — если изучена только схема БД
  - `partial` — если изучена схема + часть бизнес-логики или кода
  - `full` — если изучена схема + бизнес-логика + код + формы

### Фаза 5: Финальный отчёт

Выведи отчёт в формате:

```
## Результаты исследования: "$ARGUMENTS"

### Обзор
[2-3 предложения: что это, для чего, основные таблицы]

### Структура данных
- Основная таблица: [TableName] ([кол-во колонок] полей, [кол-во записей] записей)
- Дочерние таблицы: [DetailTable1], [DetailTable2]
- Ключевые FK: [перечислить основные связи]

### Бизнес-логика
- Типы/статусы: [перечислить если есть]
- Основные операции: [создание, редактирование, удаление, ...]
- Особенности: [gotchas если есть]

### Код
- Форма: [ClassName] -> [FileName]
- SQL: [SQLFileName]
- Бизнес-класс: [ClassName] (если есть)

### Покрытие (до -> после)
| Сущность | Было | Стало | Сохранено |
|----------|------|-------|-----------|
| EntityA  | unknown | full | 4 facts, 2 rel, 1 glossary |
| EntityB  | unknown | schema_only | 1 fact |

### Что НЕ исследовано (для будущих задач)
- [перечислить если осталось что-то за рамками]
```

## Ограничения

- **Максимум 3 целевых сущности** за один вызов — иначе перегрузка контекста
- **Максимум 3-4 файла C++** для чтения — цель обзор, не deep dive
- **Не исследовать** сущности с покрытием `full` — уже изучены
- **Не создавать** SQL-запросы — только читать существующие
- **Всегда сохранять** результаты, даже если найдено мало

## Примеры

### Пример 1: Исследование по имени таблицы
```
/pm-research Orders
-> Фаза 1: ask_business -> 2 факта, 3 relationships, coverage=unknown
-> Фаза 2: get_table_schema, get_table_relations, get_table_sample
-> Фаза 3: Grep *Dlg.h, *ViewForm*.h, *_SQL.h -> найдены классы
-> Фаза 4: bulk_save(5 facts, 1 glossary, 4 relationships, 1 form_mapping)
-> Фаза 5: Отчёт, coverage: unknown -> partial
```

### Пример 2: Исследование по бизнес-термину
```
/pm-research обработка заказов
-> Фаза 1: ask_business -> сущность Orders
-> Фаза 2: dbmcp: schema + relations + sample
-> Фаза 3: Grep для форм и SQL
-> Фаза 4: Сохранение
-> Фаза 5: Отчёт
```

### Пример 3: Сущность уже изучена
```
/pm-research Products
-> Фаза 1: ask_business -> coverage=full, 12 фактов
-> Фаза 5: "Products уже полностью изучена. Дополнительное исследование не требуется."
```

## Связанные команды

- `/pm-lookup [запрос]` — только чтение из project-memory (без исследования)
- `/pm-remember` — curated сохранение evidence-backed итогов исследования
- `/pm-research [тема]` — полный цикл: поиск -> исследование -> сохранение -> отчёт
