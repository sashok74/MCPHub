# /ui-test — Универсальное UI-тестирование через MCP

Запускает агент `ui-test` для выполнения тестового сценария на работающем приложении.

## Аргумент

Свободный текст — описание тестового сценария на естественном языке.

```
/ui-test открыть список заказов, найти заказ с максимальной суммой, проверить что кнопка "Редактировать" активна
/ui-test на форме перемещений выбрать склад "Основной", проверить грид
/ui-test snapshot
```

## Инструкция

Используй инструмент **Task** для запуска агента `ui-test`:

```
Task(
  subagent_type="ui-test",
  prompt="Выполни тестовый сценарий: $ARGUMENTS",
  description="UI test: <краткое описание>"
)
```

Передай полный текст аргумента в prompt агента. Агент самостоятельно:
1. Соберёт контекст из project-memory (формы, контролы, бизнес-правила)
2. Выполнит шаги сценария через MCP UI-инструменты
3. Проверит результаты через UI snapshot и/или dbmcp
4. Вернёт отчёт с PASS/FAIL для каждого шага
5. Сохранит обнаруженные новые знания в project-memory

После получения результата от агента — выведи пользователю краткую сводку результатов.
