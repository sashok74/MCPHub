# Skill: db-query

Скил для работы с базой данных через MCP dbmcp.
Используй этот скил когда нужно проверить структуру базы данных, найти таблицы/колонки или выполнить SQL-запрос.

## Когда использовать

- Пользователь просит проверить структуру таблицы в БД
- Нужно найти таблицу или колонку по имени/паттерну
- Требуется выполнить SELECT-запрос к базе данных
- Нужно получить примеры данных из таблицы
- Проверка наличия данных, связей между таблицами
- Анализ схемы БД для планирования изменений в коде

## Доступные инструменты MCP dbmcp

### 1. `mcp__dbmcp__list_databases`
Список всех баз данных на сервере.

### 2. `mcp__dbmcp__list_tables`
Список таблиц. Параметры:
- `schema` (опционально) — фильтр по схеме
- `include_views` (опционально) — включить представления

### 3. `mcp__dbmcp__get_table_schema`
Полная структура таблицы (колонки, типы, PK, FK, индексы, default-значения).
- `table` — имя таблицы, например `dbo.Employees`

### 4. `mcp__dbmcp__get_table_sample`
Примеры строк из таблицы (TOP N).
- `table` — имя таблицы
- `limit` — количество строк (по умолчанию 5, макс. 100)

### 5. `mcp__dbmcp__search_columns`
Поиск колонок по паттерну во всех таблицах (SQL LIKE).
- `pattern` — паттерн поиска, например `%Email%`, `%Price%`

### 6. `mcp__dbmcp__execute_query`
Выполнение произвольного SQL-запроса.
- `sql` — SQL-запрос

## Порядок работы

1. **Поиск таблицы**: если имя таблицы неизвестно точно — используй `list_tables` или `search_columns` для поиска по имени колонки.

2. **Изучение структуры**: используй `get_table_schema` для получения полной информации о колонках, типах данных, ключах и связях.

3. **Просмотр данных**: используй `get_table_sample` для быстрого просмотра примеров данных, или `execute_query` для конкретного SELECT-запроса.

4. **Выполнение запроса**: используй `execute_query` для любых SQL-запросов. Учитывай диалект подключённой БД (MSSQL/PostgreSQL/Firebird).

## Важно

- Диалект SQL зависит от провайдера, настроенного в Database MCP модуле (MSSQL, PostgreSQL, Firebird)
- Для безопасности предпочитай SELECT-запросы
- При изменении данных (INSERT/UPDATE/DELETE) всегда уточняй у пользователя перед выполнением
