# /db-snapshots - Список сохранённых снимков запросов

Показывает все сохранённые снимки SQL-запросов с метаданными.

## Использование

```
/db-snapshots
```

## Что делает команда

1. **Получает список** всех снимков из локальной SQLite базы
2. **Показывает метаданные** каждого снимка:
   - Имя снимка
   - SQL-запрос (текст)
   - Количество строк
   - Контрольная сумма (checksum)
   - Время выполнения (мс)
   - Дата создания

## Пример вывода

```
/db-snapshots
→ Сохранённые снимки (3):

| Имя           | Строк | Checksum | Время  | Создан           |
|---------------|-------|----------|--------|------------------|
| before_opt    | 1523  | A1B2C3D4 | 245ms  | 2026-01-30 10:15 |
| after_opt     | 1523  | A1B2C3D4 | 89ms   | 2026-01-30 10:18 |
| test_query    | 42    | E5F6G7H8 | 12ms   | 2026-01-30 09:30 |

SQL-запросы:
• before_opt: SELECT * FROM IncomingOrders WHERE Status = 1...
• after_opt: SELECT io.* FROM IncomingOrders io WITH (INDEX...
• test_query: SELECT TOP 50 * FROM Storage WHERE GroupFlag...
```

## Workflow

```
1. /db-snapshot name1 SELECT ... → создать первый снимок
2. /db-snapshot name2 SELECT ... → создать второй снимок
3. /db-snapshots → просмотреть все снимки
4. /compare-snapshots name1 name2 → сравнить результаты
5. /delete-snapshot name1 → очистить ненужные
```

## Связанные команды

- `/db-snapshot` — создание нового снимка
- `/compare-snapshots` — сравнение двух снимков
- `/delete-snapshot` — удаление снимка

## Когда использовать

- **Обзор** — посмотреть какие снимки уже созданы
- **Выбор для сравнения** — определить какие снимки сравнивать
- **Очистка** — найти старые снимки для удаления
- **Анализ производительности** — сравнить время выполнения разных версий запроса
